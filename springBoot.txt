Life Cycle of the bean

@Component
public class DatabaseManager {

    public DatabaseManager() {
        System.out.println("1. Constructor: I am born, but I have no dependencies yet.");
    }

    @Autowired
    public void setSettings(AppSettings settings) {
        System.out.println("2. Injection: Spring just gave me my settings.");
    }

    @PostConstruct
    public void init() {
        System.out.println("3. PostConstruct: I am fully built. Now I will open the DB connection.");
    }

    public void doWork() {
        System.out.println("4. Service: I am doing my job now.");
    }

    @PreDestroy
    public void cleanup() {
        System.out.println("5. PreDestroy: The app is closing. I am closing the DB connection.");
    }
}

- When you use @Bean on a method, Spring uses the method name as the name of the bean in the ApplicationContext.

  @Configuration
  public class NetworkConfig {

    @Bean
    public HttpClient outboundClient() { // The bean name is "outboundClient"
        return new HttpClient();
    }
}

- if you try to @Autowire this later using a name, or if you use context.getBean("outboundClient"), Spring looks for that exact method name.


- you can also override the bean name also

  
  @Bean(name = "primaryDatabase")
   public DataSource dataSource() { 
    // Internally, Spring ignores "dataSource" and names this "primaryDatabase"
    return new MyDataSource();
   }   


- implements CommandLineRunner what does it do?

- It is a simple Functional Interface. If you create a class that implements this interface and mark it as a @Component, Spring Boot will automatically execute
  its run() method after the ApplicationContext is fully loaded and refreshed.

- We use it for tasks that need the entire system to be online before they start.

1) Database Seeding: Inserting "Admin" users or default settings into a database.

2) Cache Warming: Pre-loading data from a database into memory.

3) Validation: Checking if external APIs or files are accessible before the app starts taking traffic.

Annotations to be used:

1) @RestController - tells spring to call this method and return data in json format

2) @RequestMapping - @RequestMapping("/api/v1/books"), Usually placed at the Class level. It’s the "Base Address" of your service.

3) @PathVariable - /api/v1/books/101 method definition: public Book getBook(@PathVariable("id") Long bookId) 

4) @RequestBody - Used for sending complex objects (usually in POST/PUT).

5) Spring uses a library called Jackson under the hood to convert Java <---> JSON.

6) @JsonFormat(shape = JsonFormat.Shape.STRING, pattern = "dd-MM-yyyy HH:mm:ss")
   private LocalDateTime publishedDate;

7) @JsonProperty - Sometimes your Java variable name is userEmail, but the client wants the JSON to say email_address.

   @JsonProperty("email_address")
   private String userEmail;


8) @JsonIgnore - Crucial for security. Use this on fields like password or internalId so they never get sent in the JSON response.

9) ResponseEntity<T> - It allows you to wrap your data inside a professional envelope that included data, HttpCodes,Headers and Body.

ResponseEntity.ok(new ApiResponse<>(HttpStatus.OK, event)))
ResponseEntity.status(HttpStatus.NOT_FOUND).body(new ApiResponse<>(HttpStatus.NOT_FOUND, "Event not found")));

Q) What is an ORM?

- ORM (Object-Relational Mapping) is just a fancy way of saying "Automatic Translation."

- Mapping literally means:

  This Class = This Table.

  This Variable = This Column.

  This Object = This Row. 


- JPA (The Specification)

   In CS, a Specification is a document or a set of rules (like an Interface in Java). It defines "what" should be done, but it doesn't provide the "how."

   JPA is just a collection of Interfaces and Annotations (like @Entity, @Id, @Table).

   It defines a standard way to map a Java Class to a Relational Table.

   Crucial Point: JPA cannot run by itself. It needs a "Provider" (an Implementation) like Hibernate to actually do the work.

   Analogy: JPA is the Interface; Hibernate is the Class that implements it.


- Spring Data JPA provides the Logic.
