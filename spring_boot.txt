*** Spring Data JPA Auditing | Envers ***

- To catch any data modifications, culprits.
- who created/added this object, when this object created date, who modified/updated
  this object last time, when this object modified last time

- created_by, created_dt, modify_by, modify_dt columns in table

- inside entity, private String createdBy, private Date createdDate, private String
  lastModifiedBy, private Date lastModifiedDate


- @CreatedBy, @CreatedDate @Temporal(Temporal.TIMESTAMP), @LastModifiedBy,
  @LastModifiedDate

- How to know createdBy and modifiedBy?

- For above question, create a Class which implements AuditorAware and override
  method getCurrentAuditor(), now you can autowire its object. To use this create
  its bean

  @Bean
  public AuditorAware<String> auditorAware(){
   
   return new AuditorAwareImpl()
  }

  and to use it use annotation @EnableJpaAuditing(auditorAwareRef="auditorAware")
  just where @SpringBootApplication annotation is there.


- To populate the change in dates, we need something to happen before req reaches db

for this, over entity class use annotation @EntityListeners(AuditingEntityListener.class)


- revision part pending where tables are autogenerated for keeping track of changed
  records
