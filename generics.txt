- List cars is different from List<Car> cars

- java generics in our own classes


1) class myGeneric{

   public <T> void add(T element, List<T> list){
    	
        list.add(element);
   }

   
   public <T> T addAndReturn (T element, List<T> list){
	
	list.add(element)
        return element;
   }

}

2) now this class can be called by List and element of different types

3) here the instance of myGeneric is not bound to any type

4) to create instance of myGeneric to a particular type


   class myGenericClass<T>{

        list.add(element);
   }

	list.add(element)
        return element;
   }
}

7)  concept of generic wildcards

  List<ChildA> ca = new ArrayList<>();

  processParent(p)
  processParent(ca) -> error even though they are subclasses of parent
  processParent(cb) -> error


  public static void processParent(List<Parent> elements){

    Parent parent = elements.get(0);
    elements.add(new Parent())
}



- In order to solve this problem we have generics wildcard

  public static void processAnyParent(List<?> elements){

    Object parent1 = elements.get(0);

    Parent parent2 = elements.get(0); // we dont know type

    elements.add(new Object()) // error here, because at this time we dont know
    what type this list will have, even new Object() is not allowed
}
 

-  public static void processAnyParent(List elements){

    Object parent = elements.get(0);
